Teddy BearMuseum

Project

기존에 있는 사이트를 새롭게 리디자인하고 스크립트를 사용하여 구현

정보만 가득한 기존 웹을 보완하여, 좀 더 깔끔하고 웹에 어울릴만한 디자인으로 수정하고,정보제공에 있어 텍스트를 줄여 간단하고 정확하게 정보전달을 목적으로 두었습니다.

  • 팀명: 킹받조
  • 타입: web_redesign
  • 기여도: 팀프로젝트 100%
  • 작업기간: 23.06.30 ~ 07.14

팀 구성원에 따라 담당 업무를 정하여, 각자의 파트(메인, 서브1,2,3)를 디자인하고 서로 의 견을 통해 보완하여 수정하고,어려운 부분은 서로 도와가며 구현하였습니다.

Progress

  • 사이트 선정
  • 웹에 필요한 부분들을 선정하고 메인과 서브페이지 구성
  • 파트 분배하여 figma를 통해 디자인
  • 디자인 컨펌
  • 퍼블리싱 가이드라인 작성 / 제출
  • 공통 assets 폴더, style 파일, 변수설정, 오픈그래피, 파비콘, header, footer 작성
  • 메인 및 서브페이지 html, css 작성
  • 수정 및 스크립트 작성
  • 최종 수정
  • 최종시안 제출 및 완료보고서 작성

Problems & Solution

  • 메뉴바 슬라이드 DOWN은 되었는데 UP이 되지않는다!

  • .slideToggle()을 사용하였다.

    메뉴 하단 박스를 display: none; 으로 해두었고, .toggleClass();를 사용하여 클래스가 붙으면 display: block; 으로 구현 toggle을 사용하였기 때문에 붙었다 떨어졌다가 반복적으로 구현된다.

  • SVG를 통해 이벤트 효과를 주고싶다.

  • 텍스트만 있어 단조로움을 피하고자 이벤트를 주고싶다.

    텍스트 중간 어떠한 이벤트를 줘야할지 고민을 해보고 어느 순간에 이벤트가 구현되어야 이벤트 부분을 강조할수 있는지에 대해 생각해 보았습니다.

    최종적으로는 심플한 디자인이지만, hover을 했을때 스크롤을 했을때 구현되는 svg 애니메이션으로 단조로움은 피하고, 포인트를 줄수 있어서 만족스러운 결과물 이었습니다.

잘한 점

스크립트 실력 부족으로 다양한 스크립트를 구현하지 못한거 같아 아쉬움을 느꼈습니다.

잘한 점

  • 다양한 의견을 제시하고, 서로의 의견을 존중하며 잘 조율함
  • 생각한 디자인을 각자 만들어 보고 피드백을 주고받으며 장점만 수렴
  • 처음 하는 부분이지만 기간 내에 끝까지 완성
  • 모두가 성실하고 열심히 참여

Review

스크립트를 배운후 웹페이지에 처음 적용을 해보는데, 생각했던 디자인과 스크립트가 구현되어 신기하면서, 더 많이 공부를 해야겠구나를 느꼈습니다. 또 막히는 부분도 팀원들과 힘을 합쳐 검 색하고 이야기를 나누면서 하나하나 해 나아가는걸 보면서 너무 뿌듯하고, 또 각자의 파트가 있 음에도 서로 도우며 완성해내는 모습이 너무 좋았고, 배울점이 많았습니다.

또한 git을 사용함에 있어서 혼자만 사용해 봤는데, 이번 팀 프로젝트를 하면서 Git organization을 사용하여, 팀원들과 push, pull을 통해 정보를 공유하고 충돌이 일어나면 어떻 게 해결하는지에 대해서도 배워 볼수 있는 기회였습니다.
